Frequently Asked Questions

What is Curlmate?

Curlmate is a persistent authentication layer for remote MCP servers. It securely stores refresh tokens and automatically issues fresh access tokens via API so your integrations don’t break when tokens expire.

Who is this for?

Developers running OAuth-backed remote MCP servers, especially those integrating with Claude Desktop.

What problem does it solve?

Access tokens expire. Refresh logic fails. OAuth callbacks are painful. When integrations break mid-flow, users often churn instead of debugging. Curlmate handles token refresh and auth reliability so your MCP servers stay connected.

How does it work?

You authorize a third-party service once via OAuth. Curlmate securely stores the refresh token. When your MCP server needs access, it calls the Curlmate API and receives a fresh access token.

Do I still need to implement OAuth?

No. Curlmate handles the OAuth flow and token refresh. Your MCP server only needs to request fresh access tokens via API.

What happens when access tokens expire?

Curlmate automatically uses the stored refresh token to obtain a new access token before returning it to your server.

What about refresh token rotation or scope changes?

Curlmate handles refresh token updates and stores new tokens when providers rotate them. If scopes change or authorization is revoked, you will need to re-authorize.

Is this secure?

Refresh tokens are stored securely with encryption and access controls. API access requires a valid API key and connection identifier.

Can I use this outside of Claude Desktop?

Yes. Curlmate works with any backend or remote MCP server that needs reliable OAuth token refresh via API.

What services are supported?

Support depends on configured OAuth providers. Early access currently focuses on commonly used API services for MCP integrations.

Is this production-ready?

Curlmate is in early access and focused on providing reliable token refresh for real-world MCP deployments. Remote MCP integrations are live and used as proof of reliability.

How do I get started?

Request an API key, authorize your connection once, and use the token endpoint in your MCP server to retrieve fresh access tokens.

What happens if I revoke access from the provider?

If you revoke Curlmate’s OAuth access from the third-party provider, token refresh will stop and you will need to re-authorize.

Does Curlmate call third-party APIs directly?

No. Curlmate only manages OAuth tokens. Your MCP server calls the third-party API using the access token provided.
